# 测试指南

## 当前测试状态

当前项目在 `test/protocol_test.dart` 中包含协议与服务层基础测试：

- 接收记录 JSON 持久化与恢复。
- `hello` 端口字段防御性解析。
- 收件箱时间格式化（当天与跨天）。
- 远端无响应时剪贴板发送超时。

```bash
flutter test
```

当前测试已通过，但覆盖率仍然有限。后续应继续补齐服务层和 UI 行为测试。

## 推荐测试分层

### 单元测试

适合覆盖：

- JSON payload 解析。
- 文件名清理。
- peer 过期逻辑。
- device ID 持久化。
- 错误状态转换。

### Service 测试

适合覆盖：

- HTTP `/info` 返回。
- HTTP `/clipboard` 接收。
- HTTP `/file` 保存。
- 发送端对 HTTP 错误的处理。

### Widget 测试

适合覆盖：

- 没有设备时展示搜索状态。
- 选中设备后按钮可用。
- 接收记录展示。
- 发送失败时展示错误。

### 集成测试

适合覆盖真实端到端链路：

- 同一台机器启动两个 service 实例。
- 使用不同端口模拟两台设备。
- 发送文本并确认接收端记录。
- 发送文件并校验文件内容。

## 手动测试清单

两台设备联调时：

- 同一 Wi-Fi 下能互相发现。
- 断开一台设备后 15 秒左右消失。
- 发送纯文本剪贴板成功。
- 空剪贴板时提示为空。
- 远端连接后不响应时，剪贴板发送能按超时失败。
- 发送小文件成功。
- 发送大文件（如数百 MB）不会因固定 60 秒超时而失败，超时会按文件大小放宽。
- 接收中断（发送端中途退出）后，接收目录中不残留半截文件，收件箱也没有对应记录。
- 剪贴板内容超过 10MB 时发送失败，接收端返回 413。
- Windows 上“打开接收目录”和“在资源管理器中显示”成功时提示成功而不是“操作失败”。
- 文件名包含特殊字符时能安全保存。
- 接收同名文件时不会覆盖旧文件，收件箱显示实际保存后的文件名。
- discovery 收到非法端口、0 端口或超过 65535 的端口时不会加入在线设备列表。
- 防火墙拒绝时显示发送失败。
- macOS/Windows 窗口失焦、隐藏或最小化后，远端退出仍能在 15-20 秒内刷新在线数。
- 接收端重启后设备 ID 保持稳定。
- 接收端重启后收件箱历史仍显示之前收到的文本和文件。
- 收件箱超过 20 条记录时，点击“显示更多”能继续展开历史记录。
- 设备清单发现远端设备后，顶部右上角在线数与设备清单数量一致。
- 非当天的收件箱记录显示 `MM-DD HH:mm`。

- `/clipboard` 接收测试。

- 发送端 HTTP 错误测试。
- 大文件失败测试。
- UI 状态测试。

优先级 P2：

- 多 peer 排序测试。
