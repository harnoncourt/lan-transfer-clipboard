# 测试指南

## 当前测试状态

当前项目包含一个基础 Flutter 测试：

```bash
flutter test
```

当前测试已通过，但覆盖率很低。后续应补齐协议、服务层和 UI 行为测试。

## 推荐测试分层

### 单元测试

适合覆盖：

- JSON payload 解析。
- 文件名清理。
- peer 过期逻辑。
- device ID 持久化。
- 错误状态转换。

### Service 测试

适合覆盖：

- HTTP `/info` 返回。
- HTTP `/clipboard` 接收。
- HTTP `/file` 保存。
- 发送端对 HTTP 错误的处理。

### Widget 测试

适合覆盖：

- 没有设备时展示搜索状态。
- 选中设备后按钮可用。
- 接收记录展示。
- 发送失败时展示错误。

### 集成测试

适合覆盖真实端到端链路：

- 同一台机器启动两个 service 实例。
- 使用不同端口模拟两台设备。
- 发送文本并确认接收端记录。
- 发送文件并校验文件内容。

## 手动测试清单

两台设备联调时：

- 同一 Wi-Fi 下能互相发现。
- 断开一台设备后 15 秒左右消失。
- 发送纯文本剪贴板成功。
- 空剪贴板时提示为空。
- 发送小文件成功。
- 文件名包含特殊字符时能安全保存。
- 防火墙拒绝时显示发送失败。
- 接收端重启后设备 ID 保持稳定。
- 接收端重启后收件箱历史仍显示之前收到的文本和文件。
- 收件箱超过 20 条记录时，点击“显示更多”能继续展开历史记录。
- 设备清单发现远端设备后，顶部右上角在线数与设备清单数量一致。

## 建议补充的自动化测试

优先级 P0：

- `_safeFileName` 行为测试。
- `/clipboard` 接收测试。
- `/file` 接收测试。
- peer pruning 测试。

优先级 P1：

- 发送端 HTTP 错误测试。
- 大文件失败测试。
- UI 状态测试。

优先级 P2：

- 多 peer 排序测试。
- 接收记录顺序测试。
- 不合法 UDP packet 忽略测试。
