# 路线图

## Phase 0：MVP 骨架

状态：已完成基础实现。

范围：

- Flutter 项目骨架。
- UDP 广播发现。
- HTTP 传输服务。
- 剪贴板发送。
- 文件发送。
- 接收记录。
- 项目文档。

## Phase 1：可用性补强

目标：让工具在可信局域网内更稳定、更好用。

任务：

- 生成平台目录并补齐平台权限。
- 增加传输进度。
- 增加取消传输。
- 增加发送失败重试。
- 增加接收文件打开入口。
- 增加接收目录入口。
- 增加设备别名。
- 补齐单元测试和 service 测试。

## Phase 2：安全生产化

目标：让工具可以在半可信网络中使用。

任务：

- 设备配对。
- 信任设备列表。
- 未配对设备拦截。
- 接收前确认。
- 文件大小限制。
- 剪贴板长度限制。
- 请求签名。
- payload 加密。
- 文件 hash 校验。

## Phase 3：平台体验

目标：让各平台体验接近原生工具。

任务：

- macOS 菜单栏入口。
- Windows tray 入口。
- Android 前台服务。
- iOS 前台接收优化。
- 本地通知。
- 深色模式细节。
- 多语言。

## Phase 4：高级传输

目标：支持更复杂的局域网传输场景。

任务：

- 文件夹传输。
- 多文件队列。
- 分片上传。
- 断点续传。
- 局域网测速和最佳路径选择。
- mDNS/Bonjour 发现。
- 手动 IP 添加设备。

## Phase 5：产品发布

目标：面向普通用户分发。

任务：

- 图标和品牌。
- 安装包签名。
- macOS notarization。
- Windows installer。
- Android AAB。
- iOS TestFlight。
- 隐私政策。
- 用户帮助文档。
